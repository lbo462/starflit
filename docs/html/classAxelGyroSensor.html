Detailed Description

It's used to control the Accelerometer and the Gyroscope of the robot.

AxelGyro stands for Accelerometer + Gyroscope. One should call setup() once and update() at each frame

unsigned long  maxDelay = 10000

Calibrate the sensors.

This function blocks execution while the calibration is not over and the timer not elapsed. It should be called once during setup(). This functions sets values to the offset vectors.

Parameters
allowedAccelErrorError vector allowed for acceleration calibration
allowedGyroErrorError vector allowed for the gyroscope calibration
maxDelayMax amount of milliseconds to wait before stopping calibration. This functions returns 1 if the calibration process was stopped by this timer, meaning the calibration couldn't find offset within allowed errors.
Returns
True if the calibration could be done, false otherwise

Now, we verify that the value we read match the allowed errors If this verification fails, we update the offset with the mean If the verification passes, consider the sensor ready and calibrated!

Get the mean values of the sensor.

This function is supposedly called when the sensor isn't moving. This is called during calibration. Note that the sensors reading do not take offset into computations!

Parameters
bufferSizeAmount of measures to take for the mean, default is 100. Making it bigger makes it more precise but slower as well.

Vector3D AxelGyroSensor::complementaryFilterOutput = Vector3D()

Get the angle speed of the sensor through the complementary filter Require that the AxelGyro instance was updated beforehand through the update() method.

The x-coordinate is the pitch, The y-coordinate is the roll, The z-coordinate is the yaw. Its name is complementaryFilterOutput until we find a better name.
